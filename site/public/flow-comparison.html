📊 Resumo — 2026-06-10

✅ Implementado nesta sessão (7 features)

  • Receita Federal obrigatória no aceite de proposta (nome + situação)
  • QR Code PIX real (lib qrcode JS, 220×220)
  • Polling automático 5s (sem botão "Já paguei")
  • Score de risco visível na UI (pedido + disputas)
  • Upload de evidências de disputa (fotos/vídeos base64)
  • Contestação real do vendedor (API + contra-evidências)
  • Endpoint /kyc/document/{doc} (cache 24h em memória)

🟣 Exclusivo WhatsApp (por design)

  • Off-ramp automático (chave PIX coletada no início)
  • Notificação push ao comprador
  • Resolução por IA (OpenAI)
  • Rastreio proativo (push a cada evento Correios)
  • Fluxo conversacional guiado
  • Score de risco na notificação de custódia

🔵 Exclusivo Site (por design)

  • Histórico visual de pedidos
  • Wallet com saldo disponível / em custódia
  • Estimativa de frete por CEP
  • Dashboard de métricas do vendedor
  • Página de conclusão com recibo

⚠ Diferem — aceito como trade-off de canal

  • Proposta: WA é push; site é pull (link aberto)
  • Disputa: WA tem 5 razões; site usa texto livre
  • Confirmação: WA é 1 toque; site tem alert dialog
  • Rastreio: WA proativo; site sob demanda
  • Off-ramp: WA automático; site manual

Fluxo core (ambos os canais): CPF verificado Receita Federal → QR Code PIX → confirmação automática via webhook → custódia Soroban → comprador confirma entrega → liberação → off-ramp PIX.
Os dois canais compartilham o mesmo APICash backend, Postgres, Stellar/Soroban e Gatebox.
