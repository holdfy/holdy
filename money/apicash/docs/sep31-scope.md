# SEP-31 (cross-border) — scope

**SEP-31** flows (sending/receiving anchors for cross-border value movement) are **not implemented** in this repository today.

Integration work starts when:

1. Licensed anchor partners expose **official SEP-31** (or equivalent documented HTTP) endpoints for your corridor.
2. Product/legal scope for sender–receiver jurisdictions is approved.

Until then, APICash remains focused on **national PIX ↔ anchor HTTP** (`AnchorClient`) plus **Stellar/Soroban custody** as documented elsewhere.
