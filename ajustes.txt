### 2.7 Importador Universal de Produtos

**Conceito:** Vendedor envia qualquer link de produto — OLX, Mercado Livre, Instagram, TikTok Shop, Facebook Marketplace, WhatsApp Business, loja desconhecida, e-commerce próprio, qualquer URL — via WhatsApp, API ou Site. A Holdfy extrai automaticamente título, preço, fotos e descrição e gera um link de pagamento protegido. Sem re-cadastro.

**Por que faz sentido**
- Holdfy não é plataforma de anúncio: é camada de pagamento protegido em cima de onde o vendedor já vende.
- Fluxo natural no Brasil: vendedor negocia no WhatsApp, manda qualquer link, recebe link de escrow.
- Não depende de lista de plataformas suportadas: funciona com qualquer URL da internet.
- Diferencial direto: nenhum concorrente de escrow faz isso de forma universal.

**Por que funciona com qualquer URL**

A maioria dos sites de e-commerce implementa dados estruturados para SEO. O extrator opera em camadas:

| Camada | Técnica | Cobertura |
|---|---|---|
| 1 | `schema.org/Product` JSON-LD no HTML | Maioria dos e-commerces modernos, OLX, Shopee, ML |
| 2 | Open Graph meta tags (`og:title`, `og:image`) | Instagram, Facebook, TikTok, qualquer site com OG |
| 3 | Parser específico por plataforma | Mercado Livre (API oficial), OLX — precisão garantida |
| 4 | LLM fallback (Claude) | Qualquer página restante — manda HTML, extrai dados via IA |

A combinação cobre literalmente qualquer URL pública. O vendedor sempre confirma preço e fotos antes de gerar o link (dado pode estar desatualizado).

**Dados extraídos**
- Título do produto
- Preço sugerido (vendedor confirma / ajusta)
- Fotos (re-hospedadas internamente — nunca usar URL externa direta)
- Descrição

**Componente técnico: `apicash-importer`**
- Trait `Extractor` com pipeline em camadas (JSON-LD → OG → parser específico → LLM)
- Downloader de imagens com re-hospedagem interna
- Headless browser (Chromium) para páginas com JS dinâmico
- Fila de processamento via Pulsar
- Cache de resultado no Redis (TTL ~5 min)
- Sandbox de URL (validação, bloqueio de IPs internos, timeout)

**Fluxo WhatsApp**
1. Vendedor envia qualquer URL no WhatsApp
2. Bot detecta que é um link e aciona `apicash-importer`
3. Extrator passa pelas camadas até obter dados
4. Bot exibe preview: título, preço, foto principal → "Confirma? Pode ajustar o preço."
5. Vendedor confirma ou corrige o preço
6. Holdfy cria transação + gera link de pagamento protegido
7. Bot entrega link para vendedor compartilhar com comprador

**Fluxo Site**
1. Campo "Cole o link do produto (qualquer plataforma)"
2. Auto-preenchimento do formulário com dados importados
3. Vendedor revisa e ajusta preço/descrição se necessário
4. Confirma → gera link de checkout Holdfy

**Fluxo API**
- `POST /v1/listings/import`
- Body: `{ "url": "https://...", "seller_id": "..." }`
- Response: dados extraídos do produto + link de pagamento gerado
- Parceiros e integradores podem automatizar importação em massa

## API NFS-e — Consulta CPF/CNPJ (nfse.gov.br)

Referência de curl para consultar dados de pessoa física (CPF) e jurídica (CNPJ) via portal NFS-e.
Usado pelo site para validar e auto-preencher dados no cadastro e pagamento.

```bash
COOKIE_FILE="cookies.txt"
LOGIN_URL="https://www.nfse.gov.br/EmissorNacional/Login"

rm -f "$COOKIE_FILE"

HTML=$(curl -fsS -c "$COOKIE_FILE" "$LOGIN_URL")
TOKEN=$(echo "$HTML" | sed -n 's/.*name="__RequestVerificationToken"[^>]*value="\([^"]*\)".*/\1/p' | head -n 1)

curl -fsS -b "$COOKIE_FILE" -c "$COOKIE_FILE" \
  -X POST "$LOGIN_URL" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -H "Referer: $LOGIN_URL" \
  --data-urlencode "__RequestVerificationToken=$TOKEN" \
  --data-urlencode "Inscricao=28.805.791/0001-46" \
  --data-urlencode "Senha=Senha1234*" \
  -o /dev/null -w "HTTP: %{http_code}\n"

# Consulta CPF (pessoa física)
curl -fsS -b cookies.txt \
  "https://www.nfse.gov.br/emissornacional/api/EmissaoDPS/RecuperarInfoInscricao/{CPF}?data=$(date +%Y-%m-%d)" \
  -H "Accept: application/json" \
  -H "Referer: https://www.nfse.gov.br/EmissorNacional/"

# Consulta CNPJ (pessoa jurídica)
curl -fsS -b cookies.txt \
  "https://www.nfse.gov.br/emissornacional/api/EmissaoDPS/RecuperarInfoPessoaJuridicaTomador/{CNPJ}?data=$(date +%Y-%m-%d)" \
  -H "Accept: application/json" \
  -H "Referer: https://www.nfse.gov.br/EmissorNacional/"
```

Endpoints:
- CPF:  `GET /emissornacional/api/EmissaoDPS/RecuperarInfoInscricao/{cpf}?data=YYYY-MM-DD`
- CNPJ: `GET /emissornacional/api/EmissaoDPS/RecuperarInfoPessoaJuridicaTomador/{cnpj}?data=YYYY-MM-DD`

Auth: sessão por cookie obtida via POST login com `__RequestVerificationToken` (CSRF).
Variáveis de ambiente: `NFSE_INSCRICAO` (CNPJ do emissor), `NFSE_SENHA`.
